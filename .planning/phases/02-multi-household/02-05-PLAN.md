<objective>
Complete the multi-household UX so HOUSE-02 works end-to-end in the UI: make "create another cookbook" and "join by code" reachable at any time (not only when the user has zero households), and let an admin rename a cookbook.

Context: Phase 2's signup hook auto-creates a cookbook for every user, so `household-settings-content.tsx` (`household ? <HouseholdView/> : <NoHouseholdView/>`) means the Create/Join forms in `NoHouseholdView` are now unreachable. The forms + the `createHousehold`/`joinHousehold` mutations already exist and are multi-membership-aware (02-02/02-04) — this plan surfaces them and adds rename.
Output: a `households.rename` mutation + repo fn, the create/join forms surfaced via a reusable modal triggered from the navbar switcher, an admin rename control on the household settings page, and i18n across all 11 locales.
</objective>
