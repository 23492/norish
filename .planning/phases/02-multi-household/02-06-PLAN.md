<objective>
Add a longer-lived, regenerable, shareable invite LINK (`/join/<token>`) alongside norish's existing short-lived 10-minute join code, so a cookbook admin can invite people with a URL. Scope is the SAME security model as the join code (logged-in users join; new users go through the existing signup which still respects `registration_enabled`). Invite-as-registration-bypass is explicitly DEFERRED (INVITE-02, a separate decision).

Output: an `invite_token` on households + migration; admin generate/regenerate; a PUBLIC token→cookbook-name lookup; a join-by-token mutation reusing the multi-membership join logic; an invite-link UI in Household settings; the `/join/[token]` page; i18n in all 11 locales.
</objective>
