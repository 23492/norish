# Phase 05 Plan 04-01: AssemblyAI native transcription provider + admin-UI config

<context>
norish ALREADY ships the full video-import pipeline (yt-dlp+ffmpeg download → audio → transcribe → caption-aware AI extraction) and a provider-dispatch `switch` in `transcribeAudio()`. Existing transcription providers: openai/groq/azure (Whisper via the Vercel AI SDK), generic-openai (local OpenAI-compatible), ollama (native), disabled. There is NO `assemblyai` anywhere in source today (verified by exhaustive grep) — it exists only as a fragile live boot-patch.

This plan adds AssemblyAI as a first-class, admin-UI-selectable transcription provider whose API key is read from `video_config.transcriptionApiKey` (encrypted in the DB, per SETUP-05 — NEVER an env var). AssemblyAI is NOT Whisper/AI-SDK-compatible: it has its own upload→request→poll REST flow, so it gets a bespoke `transcribeWithAssemblyAI` fn + a `case "assemblyai"` rather than the shared `experimental_transcribe` path. The proven flow is ported from `/opt/norish/camofox-patch/apply-patch.mjs` (READ-ONLY reference, lines ~194-252).

SCOPE GUARD: do NOT re-architect the pipeline, processor-factory, or download layer. AssemblyAI is a CLOUD provider (needs an API key) but has NO endpoint field and NO dynamic model listing.

TWIN-FILE GUARD: `TranscriptionProviderSchema` lives in `packages/config/src/zod/server-config.ts` (the canonical copy that every consumer imports via `@norish/config/zod/server-config`) AND a byte-identical, separately git-tracked twin at `packages/shared/src/contracts/zod/server-config.ts` (different inode, NOT a hardlink). Both MUST be edited identically or a future re-base drifts.

MODEL-FIELD GUARD: `VideoConfigSchema.transcriptionModel` is `z.string().min(1)` — REQUIRED for every enabled provider, and the admin form renders a model text-input for any provider not in `_WITH_MODEL_LISTING`. AssemblyAI has no Whisper model; map `transcriptionModel` to AssemblyAI's `speech_model` (accepts `"best"`/`"nano"`) — send `speech_model` only when the value is one of those, else omit (AssemblyAI default). This satisfies the required-model UI without breaking transcription.
</context>

The proven AssemblyAI flow (port; READ-ONLY ref /opt/norish/camofox-patch/apply-patch.mjs ~194-252):
  upload POST https://api.assemblyai.com/v2/upload {authorization: apiKey, content-type: application/octet-stream}, body=createReadStream(audioPath), duplex:"half", AbortSignal.timeout(300000) -> {upload_url}
  create POST .../v2/transcript JSON {audio_url, language_detection:true, [speech_model]}, timeout 30000 -> {id}
  poll: deadline=now+600000; loop: sleep 3000; GET .../v2/transcript/{id} {authorization: apiKey} timeout 30000; completed -> validateTranscript(data.text); error -> aiError(data.error); deadline -> aiError("timed out")

**Task 2: Implement `transcribeWithAssemblyAI` + wire the dispatch case**

<read_first>
- packages/api/src/ai/transcriber.ts (the header doc, the helpers logStart/validateTranscript/getAudioFormat, transcribeWithOllama as the fetch+aiError pattern, the transcribeAudio switch ~303-360)
- /opt/norish/camofox-patch/apply-patch.mjs lines 194-252 (the proven upload→create→poll flow — READ-ONLY, do not modify)
</read_first>

- Add `async function transcribeWithAssemblyAI(audioPath: string, apiKey: string, model: string): Promise<AIResult<string>>` just before `transcribeWithOpenAI`. Port the boot-patch flow but match THIS file's contract:
  - `logStart("AssemblyAI", audioPath, model || "best")`.
  - `const base = "https://api.assemblyai.com/v2"`.
  - Upload via `fetch(base+"/upload", { method:"POST", headers:{ authorization: apiKey, "content-type":"application/octet-stream" }, body: createReadStream(audioPath), duplex:"half", signal: AbortSignal.timeout(300000) })`. TS note: Node's RequestInit lacks `duplex`; cast the init object `as RequestInit & { duplex: "half" }` (or `as never` on the option) to satisfy `tsc` — do NOT use `@ts-expect-error` without a comment. On `!res.ok` return `aiError(\`AssemblyAI upload error: \${res.status} - \${text}\`, "PROVIDER_ERROR")`.
  - Create transcript: body `{ audio_url, language_detection: true, ...(model === "best" || model === "nano" ? { speech_model: model } : {}) }`, 30s timeout. On `!res.ok` → `aiError(... , "PROVIDER_ERROR")`.
  - Poll: `const deadline = Date.now() + 600000;` loop while `Date.now() < deadline`: `await new Promise((r) => setTimeout(r, 3000));` then GET; `!res.ok` → aiError; `data.status === "completed"` → `validateTranscript(data.text)`; `data.status === "error"` → `aiError(\`AssemblyAI transcription failed: \${data.error}\`, "PROVIDER_ERROR")`. After the loop → `aiError("AssemblyAI transcription timed out", "TIMEOUT")` (use a code that exists in the result-types union; if "TIMEOUT" is not a valid code, use "PROVIDER_ERROR").
  - Type the JSON reads with small local interfaces (e.g. `{ upload_url: string }`, `{ id: string }`, `{ status: string; text?: string; error?: string }`) — no `any`.
- Add `case "assemblyai": return await transcribeWithAssemblyAI(audioPath, apiKey, model);` after the `ollama` case in the `transcribeAudio` switch.
- Append a bullet to the file's header doc comment: `* - assemblyai: AssemblyAI upload+poll REST API (cloud, requires API key)`.
- Verify the error `code` strings used are members of the result-types union — read `packages/shared-server/src/ai/types/result.ts` first if unsure which codes exist (the file already uses `"PROVIDER_ERROR"`, `"AUTH_ERROR"`, `"EMPTY_RESPONSE"`, `"AI_DISABLED"`).
