# Phase 13 Plan 13-01: Hide the name label in the mobile bottom-nav profile item (avatar only)

<context>
On MOBILE, the bottom navigation bar's profile/account pill (right of the Recepten/Boodschappen/Kalender pill) renders a text label next to the avatar. That label is the active-cookbook name (`activeCookbookName` in `apps/web/components/navbar/mobile-nav.tsx`), which for a personal/household cookbook named after the user shows as the user's name (e.g. "Kiran Knopper…", truncated). The avatar itself is rendered by the shared `NavbarUserMenu` (trigger="avatar") and opens the profile/account dropdown.

TASK: hide ONLY that visible text label in the mobile bottom bar so the pill shows just the avatar. Keep the avatar, its tap target (it opens the menu), and its accessibility (the avatar button already has `aria-label="Open user menu"` and the HeroUI Avatar carries `name={user?.name||user?.email||"U"}`). Desktop nav (`navbar.tsx`) renders only `<NavbarUserMenu />` with NO name label — unaffected. The profile dropdown's own `{user.name}` header (`navbar-user-menu.tsx`) is INSIDE the menu — must stay. The other three bottom-nav items are unaffected.

VERIFICATION OF THE SURFACE (grep, pre-change): the ONLY visible text next to the avatar in the mobile bottom bar is the `{activeCookbookName}` span (mobile-nav.tsx lines 135-140). `user.name` appears only in (a) navbar-user-menu.tsx line 76 as the Avatar `name` prop (initials/a11y, not visible text) and (b) line 107 inside the DropdownMenu header (the profile menu itself). mobile-nav.tsx is `md:hidden` => mobile-only.
</context>
