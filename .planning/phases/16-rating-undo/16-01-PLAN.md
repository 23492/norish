# Phase 16 — Rating Undo — PLAN

**Requested by Kiran:** "Maak een manier om de beoordeling van een recept ongedaan te maken als gebruiker. Voor als je per ongeluk hebt geklikt."

## Problem
norish's `recipe_ratings.rating` is a `notNull` integer and the repo had no delete path — `setRating(0)` cannot clear a rating. A user who taps a star by accident is stuck with it.

## Approach (4 tasks)
1. **Backend** — add `removeUserRating(userId, recipeId)` (delete scoped to the caller) + `ratings.removeRating` mutation (emit `ratingUpdated`) + `RatingRemoveInputSchema`.
2. **Hook** — `useRatingsMutation().removeRating` with optimistic clear + query invalidation.
3. **UI** — `StarRating` clear affordance (click-current-star toggles off + a × button) wired into both recipe pages + i18n.
4. **Tests** — router test + real-Postgres repo isolation test (adversarially verified) + web hook tests.

## Acceptance gates
- Only the caller's own rating can be removed (cross-user isolation, adversarially proven).
- Average + count refresh after removal.
- typecheck + i18n:check + lint clean; all ratings suites green.
