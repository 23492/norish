## Deviations from Plan

**[Rule 1/3 — blocker on the plan's own build gate] Ported permission-policy-card.tsx off HeroUI v2 API.**
- Found during: Task 2 build gate. Once `@heroui-pro/react` resolved, the Next.js build surfaced `Export SelectItem doesn't exist` then `Export CardBody doesn't exist` from `@heroui/react` in `apps/web/app/(app)/settings/household/components/permission-policy-card.tsx`.
- Root cause: this file (a multi-household addition) was written against HeroUI **v2** API (`SelectItem`, `CardBody`, `CardHeader`, `classNames` prop) but the fork runs `@heroui/react@^3.1.0`. It was never build-validated because (a) `tsc` resolves the `@heroui/react` import as `any` so typecheck missed it, and (b) the earlier `@heroui-pro/react` resolution failure short-circuited every prior build.
- Fix: ported to the established v3 compound already used in 5+ files — `Select` + `Select.Trigger/Value/Indicator` + `Select.Popover`/`ListBox`/`ListBox.Item`/`ListBox.ItemIndicator` (`selectedKeys`→`selectedKey`, key-set handler → single key; `classNames.trigger` → `Select.Trigger className`), and `Card`→`Card.Header`/`Card.Content`.
- Scope check: a static scan confirmed this is the **only** file appwide importing any v2-style HeroUI compound-child export — not an app-wide migration.
- Verification: web typecheck EXIT 0, eslint 0/0, build EXIT 0.
- Commit: `4bb985ed`.

**Total deviations:** 1 auto-fixed (Rule 1/3 blocker). **Impact:** unblocked the build gate; behavior preserved (identical permission dropdowns), one stray file brought onto the free HeroUI v3 API.
