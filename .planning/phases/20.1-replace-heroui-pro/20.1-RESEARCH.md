# Phase 20.1 — Replace @heroui-pro/react — RESEARCH

Researched 2026-07-15 (Claude + 21st.dev MCP). Verified against the installed tree on `integ/upstream-0.19.0`.

## 1. Key discovery: most "Pro" components have free siblings already installed

Free `@heroui/react@3.1.0` (MIT, already a dep) ships in `dist/components/`: `drawer`, `modal`, `toggle-button`, `toggle-button-group`, `toolbar`, `tabs`, … Free HeroUI v3 is built on `react-aria-components` (already installed transitively), which itself ships `DropZone` + `FileTrigger`. And `embla-carousel-react@^8.6.0` is ALREADY in `apps/web/package.json` (Pro's Carousel is embla-based, so upstream carried the dep). Consequence: **only the Carousel needs new local code; everything else is an API swap to packages already present.**

## 2. Replacement mapping (per pro primitive)

### 2.1 `Segment` → free `ToggleButtonGroup` — ✅ APPLIED (working tree)

Proven pattern already in the codebase: `apps/web/components/recipes/measurement-system-selector.tsx` uses free `ToggleButtonGroup`/`ToggleButton` as a single-select segmented control. Swap applied to `recipe-view-mode-toggle.tsx`:
- `selectedKey={k}` → `selectedKeys={[k]}` + `selectionMode="single"` + `disallowEmptySelection`
- `onSelectionChange(key)` → `onSelectionChange(keys: Set<Key>)` — destructure first element
- `Segment.Item id=…` → `ToggleButton id=…`; `Segment.Separator` → `ToggleButtonGroup.Separator` (place in 2nd..nth buttons)
- 21st.dev cross-check: component 15413 "HeroUI Toggle Button Group" (`21st.dev/@larsen66/heroui-toggle-button-group`) confirms the free-package import `import { ToggleButtonGroup, ToggleButton } from "@heroui/react"` and the `selectionMode` API.
Verified: web `tsc --noEmit` EXIT 0.

### 2.2 `Sheet` → free `Drawer` (Panel.tsx — the highest-leverage swap)

`Panel.tsx` is the app's single bottom-sheet wrapper (13 importers: mini-calendar, mini-groceries, mini-recipes, filters, recurrence, edit-grocery/tag/note/planned-recipe, add-grocery, store-manager, recipe-share-panel, panel.test).

Pro API used (Panel.tsx:145-169): `Sheet.Root`/`Sheet.NestedRoot`, `Sheet.Backdrop variant=…`, `Sheet.Content`, `Sheet.Dialog`, `Sheet.Handle`, `Sheet.CloseTrigger`, `Sheet.Header`, `Sheet.Heading`, `Sheet.Body`, `Sheet.Footer`.

Free `Drawer` compound (verified in `node_modules/@heroui/react/dist/components/drawer/index.d.ts`): `Drawer.Root` (`state` prop), `Drawer.Trigger`, `Drawer.Backdrop` (`isDismissable`, `variant`), `Drawer.Content` (`placement` — use `"bottom"`), `Drawer.Dialog`, `Drawer.Handle`, `Drawer.CloseTrigger`, `Drawer.Header`, `Drawer.Heading`, `Drawer.Body`, `Drawer.Footer` — a 1:1 slot match for everything Panel uses **except `Sheet.NestedRoot`**.

Nested-panel gap: Panel's `nested` prop (used by edit-planned-recipe-panel/edit-note-panel flows) selects `Sheet.NestedRoot`. Free Drawer has no `NestedRoot`; nested = a second `Drawer.Root` rendered while the first stays open. Risk: stacking/inert behavior differs (react-aria modal overlays stack fine, but the parent sheet won't get the pro "scale-back" effect). Acceptance: nested open/close/dismiss order works; cosmetic scale-back is NOT required for parity.

### 2.3 `Carousel`/`useCarousel` → local `apps/web/components/ui/carousel.tsx` on embla

Three call sites and their exact pro-API surface:
- `media-carousel.tsx`: `<Carousel opts={{loop:true}}>`, `Carousel.Content/.Item/.Previous/.Next/.Dots`, and `useCarousel().selectedIndex` (line 109, inside a child to gate video playback)
- `image-lightbox.tsx`: `Carousel.Content/.Item/.Previous/.Next/.Thumbnails/.Thumbnail`
- `step-images.tsx`: `<Carousel className="[--carousel-gap:0px]" opts={{loop:true}}>`, `.Content/.Item/.Previous/.Next/.Dots`

Base: **21st.dev component 813 — shadcn "Carousel"** (`21st.dev/@shadcn/components/carousel`, embla-based; full source retrieved 2026-07-15 via `get_component`, MIT). It provides `Carousel` (context provider: `carouselRef`, `api`, `scrollPrev/Next`, `canScrollPrev/Next`, `opts`, `orientation`, arrow-key handling, `role="region" aria-roledescription="carousel"`), `CarouselContent`, `CarouselItem` (`role="group" aria-roledescription="slide"`), `CarouselPrevious`, `CarouselNext`, exported `useCarousel` hook.

Required local adaptations (all straightforward on embla's public API):
1. **Compound-style aliases** so call sites keep `Carousel.Content` etc.: attach subcomponents as properties (`Carousel.Content = CarouselContent; …`) — minimizes the 3-file diff.
2. **`selectedIndex` in context**: track via `api.selectedScrollSnap()` on `select`/`reInit` events; expose from `useCarousel()` (media-carousel.tsx:109 needs it).
3. **`Carousel.Dots`**: map `api.scrollSnapList()` → buttons calling `api.scrollTo(i)`; active = `selectedIndex`. (Matches pro's `.Dots` with className passthrough.)
4. **`Carousel.Thumbnails`/`Carousel.Thumbnail`**: horizontal strip of `scrollTo(i)` buttons rendering children (images), `data-selected` on the active one (image-lightbox styling hook).
5. **Gap variable**: pro uses `--carousel-gap` (step-images sets `[--carousel-gap:0px]`); implement Content/Item spacing with `gap: var(--carousel-gap, 1rem)` instead of shadcn's `-ml-4/pl-4` so the existing className keeps working.
6. **Buttons**: replace the shadcn `Button` dep with free `@heroui/react` `Button` (`isIconOnly variant="ghost"`) — no shadcn `cn`/`cva` imports; use the repo's existing class-merge idiom.
7. Drop the retrieved demo's `Card` usage (not needed).

Rejected alternates from the 21st.dev search: 8bit/orbiting/3d carousels (styling mismatch), Reshaped carousel (own design system), ibelick/motion (motion-first, no thumbnails). shadcn base + extensions is the smallest correct fit.

### 2.4 Pro `DropZone` → `react-aria-components` `DropZone` + `FileTrigger` (local wrapper)

Pro API used (import-from-image-modal.tsx): `DropZone` (frame) → `.Area` (`getDropOperation`, `onDrop` — these are react-aria `DropZoneProps` verbatim) → `.Icon`/`.Label`/`.Description`/`.Trigger` (styled slots) + `.Input` (`multiple`, `accept`, `onSelect`).

Free path (zero new deps): `react-aria-components` `DropZone` accepts `getDropOperation`+`onDrop` with the SAME signatures (pro re-exports them — `DropZoneAreaProps["onDrop"]` maps to RAC `DropZoneProps["onDrop"]`); `FileTrigger` (`acceptedFileTypes`, `allowsMultiple`, `onSelect`) + free `Button` replaces `.Trigger`+`.Input`. Local wrapper `apps/web/components/ui/drop-zone.tsx` provides the styled compound (`Icon`/`Label`/`Description` are styled divs) so the modal diff stays minimal.
21st.dev survey (file-upload/dropzone, 8 results): all candidates are react-dropzone- or custom-hook-based → would ADD a dependency; RAC path wins. (Reference for styling only: `21st.dev/@haydenbleasel/dropzone`.)

## 3. Pitfalls

1. **Drawer `state` vs Sheet's open-prop shape** — Panel drives open state via props from 13 consumers; verify the exact controlled-state prop (`state`, or RAC `isOpen`/`onOpenChange` passthrough) against `drawer.d.ts` at implementation time and adapt inside Panel ONLY (consumers must not change).
2. **Nested sheets** (2.2) — verify edit-planned-recipe → nested note panel flow manually in the browser.
3. **`useCarousel` import site** — media-carousel imports `useCarousel` from the pro package at top level and calls it in a child of `<Carousel>`; the local hook must throw outside the provider (shadcn base already does).
4. **Chromatic/vitest snapshots** — panel.test asserts Sheet mocks; rewrite against Drawer semantics, don't port the mock.
5. **globals.css order** — pro css import sits between `@heroui/styles` and the theme import; removing it must not reorder the remaining imports.
6. **Bundle regression watch** — embla is already shipped; net dependency count goes DOWN. No license text needed beyond shadcn's MIT (add source attribution comment in carousel.tsx).
