  <name>Task 1: Rewrite packages/shared-server/src/realtime/policy.ts</name>
  <files>packages/shared-server/src/realtime/policy.ts</files>
  <action>
    (a) `emitByPolicy` â€” remove the `case "everyone": emitter.broadcast(...)` branch.
    `everyone` and `household` both route to `emitter.emitToHousehold(ctx.householdKey, ...)`;
    `owner` routes to `emitToUser`. When `ctx.householdKey === ctx.userId` (the personal-recipe
    key convention from `middleware.ts:38`) route to `emitToUser` instead, so a personal
    recipe can never land on a household channel. Document D-22-01 in the doc comment,
    including WHY `TypedRedisEmitter.broadcast` still exists for genuinely global events.

    (b) Add `resolveRecipeRealtimeScope(recipeId: string, fallback: PolicyEmitContext):
    Promise<{ viewPolicy: PermissionLevel; ctx: PolicyEmitContext }>` per D-22-02:
      - `getRecipeOwnerAndHousehold(recipeId)` from `@norish/db`;
      - null / null-owner -> FAIL CLOSED: `{ viewPolicy: "owner", ctx: fallback }`;
      - `householdId === null` -> `{ viewPolicy: (await getRecipePermissionPolicy()).view,
        ctx: { userId: owner.userId, householdKey: owner.userId } }`;
      - else `getHouseholdPolicy(householdId)`; on a missing household fall back to the
        server default policy but STILL key on the recipe's own household id;
        -> `{ viewPolicy: policy.view, ctx: { userId: owner.userId, householdKey: householdId } }`.
    Mirror the doc-comment style of `resolveRecipeCookbookPolicy` and cross-reference it.
  </action>
