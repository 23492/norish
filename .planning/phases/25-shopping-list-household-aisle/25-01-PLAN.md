# 25-01 PLAN — Shopping list: household-shared + aisle grouping (SHOP-01 + SHOP-02)

**Requirements:** SHOP-02 (household-shared list), SHOP-01 (aisle grouping)
**DB:** migration `0040` (DB 40 → 41). RE-KEYS ownership `user_id` → `household_id` on live tables.

## Context (grounded in code)
- `groceries` / `stores` / `ingredient_store_preferences` / `recurring_groceries` are `user_id`-scoped.
  "Sharing" today = router lists rows for ALL active-household member `user_id`s (`ctx.userIds`).
  Bug it causes: a user's rows follow them into EVERY household they're active in (cross-household bleed).
- The UI already groups the list by `store` (ordered aisles) and `ingredient_store_preferences` auto-assigns
  a new item to a store — so "aisle grouping" is an EXISTING affordance; SHOP-01's real content is a
  DEFAULT seed so a fresh household is grouped with zero config.
- Every user has an auto-created OWN household (admin = self) from signup. Personal view = active_household NULL.
- Authoritative schema: `packages/db-schema`. `packages/db/src/schema/*` re-exports it.

## Decisions
- **D-25-01 (row remap):** each existing row's `household_id` = that row's user's OWN household
  = earliest household `admin_user_id = user_id`, fallback earliest membership. Injective per user
  (admin_user_id is unique per household) ⇒ NO cross-user merge, NO unique-constraint collisions.
- **D-25-02 (keep user_id):** retain `user_id` as an added-by/audit column (non-destructive); all
  ISOLATION + scoping now key on `household_id`.
- **D-25-03 (personal-list home):** personal view resolves the shopping-list scope to the user's OWN
  household id (`resolveShoppingHouseholdId(ctx) = ctx.household?.id ?? ownHouseholdId(user)`).
- **D-25-04 (category seed source):** ship a BUILT-IN curated default aisle set (EN+NL common
  ingredients) — do NOT import the full open-tandoor-data dataset (heavy, licensing, out of scope).
- **D-25-05 (aisle == store level):** extend the existing `stores` + `ingredient_store_preferences`
  pattern (roadmap's "extend, don't invent"); seed default aisles as `stores`. No new tables.
- **D-25-06 (seed reach):** seed at `createHousehold` (new households) AND in migration `0040` for
  existing households that have ZERO stores. Same default set.
- **D-25-07 (realtime unchanged):** grocery/store emit stays keyed on `ctx.householdKey`
  (`household?.id ?? user.id`); audience is identical to the DB scope (personal view = the lone user).
